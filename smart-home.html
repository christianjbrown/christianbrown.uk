🏗️ How it works

This page is a static site built with Jekyll and hosted on GitHub Pages. When it loads, its JavaScript calls two APIs through the Fastly CDN, which caches the responses and forwards any misses to a pair of Google Cloud Functions. The CDN's configuration is managed as code with Terraform.

The first reads the indoor climate: it walks every device in our Samsung SmartThings account, keeps the ones that report temperature or humidity, resolves the room each belongs to, flags any reading older than 24 hours as stale, and returns them sorted by name alongside an average of the fresh readings.

The second calls the Met Office DataPoint API for our local forecast, then normalises the response — temperature and how it feels, humidity, chance of precipitation, and wind speed, gust and direction — into the compact JSON this page renders.

The moving parts are open source:

  • christianbrown.uk — the Jekyll source for this site, including the page you're reading.
  • terraform-christian-fastly — Terraform that defines the Fastly CDN configuration in front of the Cloud Functions.
  • php-gcp-function-smartthings-climate — the Google Cloud Function that serves the indoor climate data.
  • php-gcp-function-met-office-weather — the Google Cloud Function that serves the outdoor weather forecast.
  • php-smartthings-api-lib — PHP client for the Samsung SmartThings API.
  • php-met-office-weather-datahub-api-lib — PHP client for the Met Office Weather DataHub API.
  • php-gcp-function-lib — shared helpers for running the PHP Cloud Functions.
  • php-api-client-lib — the underlying HTTP API client library the above build on.
